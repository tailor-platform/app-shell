# Reconciliation

The Reconciliation components provide a complete UI for document matching workflows (e.g., invoice vs PO vs goods receipt). They are **fully configurable** — the components don't assume any specific backend schema. You define the statuses, fields, columns, tabs, and action labels through props.

There are two main components:

- **ReconciliationList** — A table with status filter tabs for browsing matching records
- **ReconciliationDetail** — A detail page showing match result metadata, discrepancies, line item comparison, related documents, and processing activity

## Import

```tsx
import {
  ReconciliationList,
  ReconciliationDetail,
  type ReconciliationListProps,
  type ReconciliationDetailProps,
  type ReconciliationListItem,
  type ReconciliationRecord,
} from "@tailor-platform/app-shell";
```

## Backend Requirements

The Reconciliation components expect the backend to provide:

### For the list page

An array of records, each with:

- `id` — unique identifier
- `status` — a string status value (you define what statuses exist)
- `data` — a flat object with all display fields (the component reads from this based on your column config)

```typescript
// Example: what your API query returns
const records = await fetchReconciliations();

// Map to the component's shape
const items: ReconciliationListItem[] = records.map((r) => ({
  id: r.id,
  status: r.status,
  data: {
    invoiceNumber: r.invoiceNumber,
    supplier: r.supplier.name,
    matchScore: r.matchScore,
    totalAmount: r.totalAmount,
    currency: r.currency,
    date: r.createdAt,
    // Add any field your columns reference
  },
}));
```

### For the detail page

A single record with:

- `id`, `status`, `matchScore` — core matching fields
- `data` — flat object for the DescriptionCard metadata section
- `summary` — text summary (also used as error message in error state)
- `processingSteps` — pipeline step objects with `id`, `label`, `status` ("completed" | "in_progress" | "pending"), and optional `timestamp`
- `discrepancies` — objects with `id`, `severity` ("warning" | "error"), `category`, `message`, `resolved`
- `lineItems` — array of flat objects (your column config defines how to render each field)
- `relatedDocuments` — objects with `id`, `type`, `label`, `status`, `statusVariant`, `autoGenerated`, `href`

## Status Configuration

Both components require a `statusConfig` object that tells the component how to interpret status values:

```tsx
const statusConfig = {
  // Which status value means "still processing" — shows loading spinner
  processingStatus: "processing",

  // Which status value means "failed" — shows error card
  errorStatus: "error",

  // Which statuses should hide the action buttons
  hideActionsForStatuses: ["matched", "processing"],

  // How each status maps to a badge variant
  badgeVariantMap: {
    matched: "subtle-success",
    partial_match: "subtle-warning",
    mismatch: "subtle-error",
    processing: "subtle-default",
    error: "subtle-error",
  },

  // Human-readable label for each status
  labelMap: {
    matched: "Matched",
    partial_match: "Partial Match",
    mismatch: "Mismatch",
    processing: "Processing",
    error: "Error",
  },
};
```

This is fully flexible — your statuses can be anything: `"pending_review"`, `"approved"`, `"rejected"`, etc.

## ReconciliationList

### Basic Usage

```tsx
const tabs = [
  { key: "all", label: "All" },
  { key: "matched", label: "Matched" },
  { key: "partial_match", label: "Partial Match" },
  { key: "processing", label: "Processing" },
];

const columns = [
  { key: "invoiceNumber", header: "Invoice #" },
  { key: "supplier", header: "Supplier" },
  { key: "status", header: "Status", type: "badge" },
  { key: "matchScore", header: "Score", type: "score", align: "right" },
  {
    key: "totalAmount",
    header: "Amount",
    type: "money",
    align: "right",
    meta: { currencyKey: "currency" },
  },
  { key: "date", header: "Date", type: "date" },
];

<ReconciliationList
  items={items}
  statusConfig={statusConfig}
  tabs={tabs}
  columns={columns}
  onItemClick={(item) => navigate(`/reconciliation/${item.id}`)}
/>;
```

### With Upload Dialog

```tsx
const [uploadOpen, setUploadOpen] = useState(false);

<ReconciliationList
  items={items}
  statusConfig={statusConfig}
  tabs={tabs}
  columns={columns}
  onItemClick={(item) => navigate(`/reconciliation/${item.id}`)}
  onUpload={async (file) => {
    const { id } = await createReconciliationJob(file);
    setUploadOpen(false);
    navigate(`/reconciliation/${id}`);
  }}
  uploadOpen={uploadOpen}
  onUploadOpenChange={setUploadOpen}
  uploadDialogProps={{
    title: "Upload Invoice",
    description: "Upload a PDF or image of a supplier invoice.",
    uploadLabel: "Upload Invoice",
  }}
/>;
```

### Column Types

| Type       | Renders                                                                    | Meta                          |
| ---------- | -------------------------------------------------------------------------- | ----------------------------- |
| `"text"`   | Plain text (default)                                                       | —                             |
| `"number"` | Right-aligned number                                                       | —                             |
| `"badge"`  | Badge with variant from `statusConfig` or column `meta.badgeVariantMap`    | `badgeVariantMap`, `labelMap` |
| `"money"`  | Currency-formatted number                                                  | `currencyKey`                 |
| `"score"`  | Color-coded percentage (green/yellow/red). Shows "—" for processing status | —                             |
| `"date"`   | Formatted date string                                                      | —                             |
| `"link"`   | Clickable link with external icon                                          | `hrefKey`                     |

### Props

| Prop                 | Type                       | Required | Description                                             |
| -------------------- | -------------------------- | -------- | ------------------------------------------------------- |
| `items`              | `ReconciliationListItem[]` | Yes      | List of records                                         |
| `statusConfig`       | `StatusConfig`             | Yes      | Status interpretation config                            |
| `tabs`               | `StatusTab[]`              | Yes      | Filter tab definitions                                  |
| `columns`            | `ListColumn[]`             | Yes      | Table column definitions                                |
| `onItemClick`        | `(item) => void`           | No       | Row click handler                                       |
| `onUpload`           | `(file: File) => void`     | No       | Upload handler. Omit to hide upload dialog              |
| `uploadOpen`         | `boolean`                  | No       | Controlled dialog open state                            |
| `onUploadOpenChange` | `(open: boolean) => void`  | No       | Dialog open state change                                |
| `uploadDialogProps`  | `object`                   | No       | Override dialog title, description, accept, uploadLabel |
| `emptyState`         | `ReactNode`                | No       | Custom empty state                                      |
| `loading`            | `boolean`                  | No       | Show skeleton loading state                             |
| `className`          | `string`                   | No       | Additional CSS classes                                  |

## ReconciliationDetail

### Basic Usage

```tsx
import type { FieldConfig } from "@tailor-platform/app-shell";

const fields: FieldConfig[] = [
  { key: "invoiceNumber", label: "Invoice Number", meta: { copyable: true } },
  { key: "supplier", label: "Supplier" },
  { key: "totalAmount", label: "Total Amount", type: "money", meta: { currencyKey: "currency" } },
  { key: "invoiceDate", label: "Invoice Date", type: "date", meta: { dateFormat: "medium" } },
  { type: "divider" },
  { key: "summary", label: "Summary", emptyBehavior: "hide" },
];

const lineItemColumns: LineItemColumn[] = [
  { key: "lineNumber", header: "#" },
  { key: "product", header: "Product" },
  {
    key: "status",
    header: "Status",
    type: "badge",
    meta: { badgeVariantMap: { matched: "subtle-success", price_mismatch: "subtle-warning" } },
  },
  { key: "invoiceQty", header: "Inv Qty", type: "number", align: "right" },
  { key: "poQty", header: "PO Qty", type: "number", align: "right" },
  { key: "priceVariance", header: "Price Var", type: "variance", align: "right" },
];

<ReconciliationDetail
  data={record}
  statusConfig={statusConfig}
  fields={fields}
  lineItemColumns={lineItemColumns}
  onCreate={() => navigate("/bills/new")}
  createLabel="Create purchase bill"
  onUpdate={() => navigate(`/po/${record.id}/edit`)}
  updateLabel="Update PO"
  onRefresh={() => refetch()}
/>;
```

### Layout by Status

The component automatically adjusts its layout based on the record's status:

| Status                                           | Left Column                                                       | Right Column                            |
| ------------------------------------------------ | ----------------------------------------------------------------- | --------------------------------------- |
| **Processing** (`statusConfig.processingStatus`) | Spinner card                                                      | Activity timeline                       |
| **Error** (`statusConfig.errorStatus`)           | Error card with summary                                           | Retry action + Activity timeline        |
| **Any other status**                             | Match Result (DescriptionCard) + Discrepancies + Line Items Table | Actions + LinkedRecordsCards + Activity |

### Line Item Column Types

| Type         | Renders                                                            | Meta              |
| ------------ | ------------------------------------------------------------------ | ----------------- |
| `"text"`     | Plain text (default)                                               | —                 |
| `"number"`   | Plain number                                                       | —                 |
| `"badge"`    | Badge with variant from `meta.badgeVariantMap`                     | `badgeVariantMap` |
| `"money"`    | Currency-formatted number                                          | `currencyKey`     |
| `"variance"` | Color-coded percentage (green for 0, yellow for <5%, red for >=5%) | —                 |

### Props

| Prop              | Type                   | Required | Description                                          |
| ----------------- | ---------------------- | -------- | ---------------------------------------------------- |
| `data`            | `ReconciliationRecord` | Yes      | The full record                                      |
| `statusConfig`    | `StatusConfig`         | Yes      | Status interpretation config                         |
| `fields`          | `FieldConfig[]`        | Yes      | DescriptionCard field config for match result        |
| `lineItemColumns` | `LineItemColumn[]`     | Yes      | Column config for line items table                   |
| `title`           | `string`               | No       | Match result section title (default: "Match Result") |
| `onCreate`        | `() => void`           | No       | Create action. Omit to hide                          |
| `createLabel`     | `string`               | No       | Create button label (default: "Create")              |
| `onUpdate`        | `() => void`           | No       | Update action. Omit to hide                          |
| `updateLabel`     | `string`               | No       | Update button label (default: "Update")              |
| `onRefresh`       | `() => void`           | No       | Polling callback during processing / retry on error  |
| `refreshInterval` | `number`               | No       | Polling interval in ms (default: 5000)               |
| `retryLabel`      | `string`               | No       | Retry button label (default: "Retry")                |
| `actions`         | `ReactNode`            | No       | Additional custom action buttons                     |
| `className`       | `string`               | No       | Additional CSS classes                               |

```tsx
// list-page.tsx
import { ReconciliationList, Layout, Button } from "@tailor-platform/app-shell";
import { STATUS_CONFIG, LIST_TABS, LIST_COLUMNS } from "./reconciliation-config";

function ReconciliationListPage() {
  const { data, loading } = useQuery(GET_RECONCILIATIONS);
  const navigate = useNavigate();

  const items =
    data?.reconciliations.map((r) => ({
      id: r.id,
      status: r.status,
      data: {
        invoiceNumber: r.invoiceNumber,
        supplier: r.supplier.name,
        matchScore: r.matchScore,
        totalAmount: r.totalAmount,
        currency: r.currency,
        date: r.createdAt,
      },
    })) ?? [];

  return (
    <Layout>
      <Layout.Header title="Invoice Reconciliation" />
      <Layout.Column>
        <ReconciliationList
          items={items}
          statusConfig={STATUS_CONFIG}
          tabs={LIST_TABS}
          columns={LIST_COLUMNS}
          loading={loading}
          onItemClick={(item) => navigate(`/reconciliation/${item.id}`)}
          onUpload={async (file) => {
            const { id } = await createJob(file);
            navigate(`/reconciliation/${id}`);
          }}
        />
      </Layout.Column>
    </Layout>
  );
}
```

```tsx
// detail-page.tsx
import { ReconciliationDetail } from "@tailor-platform/app-shell";
import { STATUS_CONFIG, DETAIL_FIELDS, LINE_ITEM_COLUMNS } from "./reconciliation-config";

function ReconciliationDetailPage() {
  const { id } = useParams();
  const { data, refetch } = useQuery(GET_RECONCILIATION, { variables: { id } });

  if (!data) return null;

  const record = {
    id: data.id,
    status: data.status,
    matchScore: data.matchScore,
    data: {
      invoiceNumber: data.invoiceNumber,
      status: data.status,
      supplier: data.supplier.name,
      totalAmount: data.totalAmount,
      currency: data.currency,
      invoiceDate: data.invoiceDate,
      summary: data.summary,
    },
    summary: data.summary,
    processingSteps: data.jobs.map((j) => ({
      id: j.id,
      label: j.description,
      status: j.status,
      timestamp: j.completedAt,
    })),
    discrepancies: data.discrepancies.map((d) => ({
      id: d.id,
      severity: d.severity,
      category: d.category,
      message: d.message,
      resolved: d.resolved,
    })),
    lineItems: data.lineItems,
    relatedDocuments: data.relatedDocuments.map((doc) => ({
      id: doc.id,
      type: doc.type,
      label: doc.number,
      status: doc.status,
      autoGenerated: doc.autoGenerated,
      href: `/${doc.type}/${doc.id}`,
    })),
  };

  return (
    <ReconciliationDetail
      data={record}
      statusConfig={STATUS_CONFIG}
      fields={DETAIL_FIELDS}
      lineItemColumns={LINE_ITEM_COLUMNS}
      onCreate={() => navigate("/bills/new", { state: record })}
      createLabel="Create purchase bill"
      onUpdate={() => navigate(`/po/${record.id}/edit`)}
      updateLabel="Update PO"
      onRefresh={() => refetch()}
      retryLabel="Retry processing"
    />
  );
}
```

## Using for a Different Workflow

The same components work for any document matching scenario. For example, a **sales order 3-way match**:

```tsx
const soStatusConfig = {
  processingStatus: "validating",
  errorStatus: "failed",
  hideActionsForStatuses: ["confirmed", "validating"],
  badgeVariantMap: {
    confirmed: "subtle-success",
    pending_review: "subtle-warning",
    rejected: "subtle-error",
    validating: "subtle-default",
    failed: "subtle-error",
  },
  labelMap: {
    confirmed: "Confirmed",
    pending_review: "Pending Review",
    rejected: "Rejected",
    validating: "Validating",
    failed: "Failed",
  },
};

const soColumns = [
  { key: "soNumber", header: "SO #" },
  { key: "customer", header: "Customer" },
  { key: "soQty", header: "SO Qty", type: "number", align: "right" },
  { key: "deliveryQty", header: "Delivery Qty", type: "number", align: "right" },
  { key: "invoiceQty", header: "Invoice Qty", type: "number", align: "right" },
  { key: "variance", header: "Variance", type: "variance", align: "right" },
];
```

The component doesn't know or care that this is a sales order — it just renders the columns you define.
